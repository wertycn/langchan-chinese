# 模板仓库

那么，您已经创建了一个非常酷的链 - 现在该怎么办呢？如何部署它并使其与世界分享起来？

本节介绍了几种选项。请注意，这些选项适用于快速部署原型和演示，而不适用于生产系统。如果您需要帮助部署生产系统，请直接联系我们。

以下是一些模板GitHub仓库的列表，旨在方便进行fork和修改以使用您的链。这个列表远非详尽无遗，我们非常欢迎在这里进行贡献。

## [Streamlit](https://github.com/hwchase17/langchain-streamlit-template)

这个仓库作为一个模板，展示了如何使用Streamlit部署LangChain。
它实现了一个聊天机器人界面。
它还包含了如何在Streamlit平台上部署该应用程序的说明。

## [Gradio（在Hugging Face上）](https://github.com/hwchase17/langchain-gradio-template)

该存储库作为一个使用Gradio部署LangChain的模板。
它实现了一个聊天机器人界面，采用“自带令牌”方法（不会产生大额费用）。
它还包含了如何在Hugging Face平台上部署此应用程序的说明。
这受到了James Weaver的[优秀示例](https://huggingface.co/JavaFXpert)的很大影响。

## [Chainlit](https://github.com/Chainlit/cookbook)

该存储库是一个使用Chainlit来可视化和部署LangChain代理的菜谱。您可以使用Chainlit创建类似ChatGPT的用户界面。其中一些关键功能包括中间步骤的可视化，元素管理和显示（图像，文本，轮播等），以及云部署。
Chainlit [doc](https://docs.chainlit.io/langchain) 上关于与LangChain集成的文档。

## [Beam](https://github.com/slai-labs/get-beam/tree/main/examples/langchain-question-answering)

该仓库作为一个模板，展示了如何使用[Beam](https://beam.cloud)部署LangChain。

它实现了一个问答应用程序，并包含了将该应用程序部署为无服务器REST API的说明。

## [Vercel](https://github.com/homanp/vercel-langchain)

一个最小化的示例，展示了如何使用Flask在Vercel上运行LangChain。

## [FastAPI + Vercel](https://github.com/msoedov/langcorn)

在Vercel上使用FastAPI和LangCorn/Uvicorn运行LangChain的最简示例。

## [Kinsta](https://github.com/kinsta/hello-world-langchain)

使用Flask将LangChain部署到[Kinsta](https://kinsta.com)的最简示例。

## [Fly.io](https://github.com/fly-apps/hello-fly-langchain)

使用Flask将LangChain部署到[Fly.io](https://fly.io/)的最简示例。

## [Digitalocean App Platform](https://github.com/homanp/digitalocean-langchain)

如何将LangChain部署到DigitalOcean App Platform的最简示例。

## [Google Cloud Run](https://github.com/homanp/gcp-langchain)

如何将LangChain部署到Google Cloud Run的最简示例。

## [SteamShip](https://github.com/steamship-core/steamship-langchain/)

此存储库包含适用于Steamship的LangChain适配器，使LangChain开发人员能够快速部署他们的应用程序在Steamship上。这包括：生产就绪的端点、跨依赖项的水平扩展、应用程序状态的持久存储、多租户支持等。

## [Langchain-serve](https://github.com/jina-ai/langchain-serve)

该存储库允许用户通过RESTful、gRPC或WebSocket API将本地链和代理作为服务，感谢[Jina](https://docs.jina.ai/)。轻松部署您的链和代理，并享受独立扩展、无服务器和自动扩展的API，以及在Jina AI云上的Streamlit游乐场。

## [BentoML](https://github.com/ssheng/BentoChain)

该存储库提供了一个示例，展示了如何使用[BentoML](https://github.com/bentoml/BentoML)部署LangChain应用程序。BentoML是一个框架，可以将机器学习应用程序容器化为标准的OCI镜像。BentoML还可以自动生成OpenAPI和gRPC端点。使用BentoML，您可以集成来自所有流行的ML框架的模型，并将它们部署为在最优硬件上独立扩展的微服务。

## [Databutton](https://databutton.com/home?new-data-app=true)

这些模板是使用Databutton构建、部署和分享LangChain应用程序的示例。您可以使用Streamlit创建用户界面，通过调度Python代码自动化任务，并在内置存储中存储文件和数据。示例包括具有对话记忆的聊天机器人界面、个人搜索引擎以及LangChain应用程序的起始模板。部署和分享只需点击一次即可。
