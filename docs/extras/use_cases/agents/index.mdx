# 代理

代理可以用于各种任务。
代理将语言模型的决策能力与工具结合起来，以创建一个能够代表您执行和实现解决方案的系统。
在继续阅读之前，强烈建议您阅读`agent`模块中的文档，以更深入地了解代理相关的概念。
具体而言，在继续阅读之前，您应该熟悉`agent`、`tool`和`agent executor`的抽象概念。

- [代理人文档](/docs/modules/agents.html)（用于与外部世界进行交互）

## 创建您自己的代理人

一旦您阅读了该文档，您就应该准备好创建您自己的代理人了。
那具体需要做些什么呢？
以下是我们推荐您开始创建您自己的代理人的步骤：

### 步骤1：创建工具

代理人的大部分功能是由它所能使用的工具定义的。
如果您有一个特定的任务要求代理人完成，您必须给它提供访问正确工具的权限。
我们在LangChain中原生地提供了许多工具，因此您应该首先查看是否有任何工具符合您的需求。
但是，我们也提供了定义自定义工具的简便方法，因此如果您需要自定义工具，绝对可以这样做。

### （可选）第2步：修改代理

内置的LangChain代理类型设计用于在通用情况下良好运行，
但是您可以通过修改代理实现来提高性能。
有几种方法可以做到这一点：

1. 修改基本提示。这可以用来为代理提供更多关于其行为方式等方面的上下文信息。
2. 修改输出解析器。如果代理在解析语言模型输出时出现问题，这是必需的。

### （可选）步骤3：修改代理执行器

通常情况下，这一步骤是不必要的，因为这是相当通用的逻辑。
您希望修改它的可能原因包括添加不同的停止条件或处理错误。
