# 提取

大多数API和数据库仍然处理结构化信息。
因此，为了更好地处理这些信息，从文本中提取结构化信息是非常有用的。
例如：

- 从一个句子中提取一个结构化的行以插入到数据库中
- 从一个长文档中提取多个行以插入到数据库中
- 从用户查询中提取正确的API参数

这项工作与[输出解析](/docs/modules/prompts/output_parsers.html)密切相关。
输出解析器负责指导LLM以特定格式进行响应。
在这种情况下，输出解析器指定了您想要从文档中提取的数据的格式。
除了输出格式指令之外，提示还应包含您想要从中提取信息的数据。

虽然普通的输出解析器对于基本的响应数据结构化已经足够好了，
但是在进行提取时，您通常希望提取更复杂或嵌套的结构。
如果您想深入了解提取的内容，我们推荐查看 [`kor`](https://eyurtsev.github.io/kor/)，
这是一个使用现有的LangChain链和OutputParser抽象的库，
但它可以深入探讨如何允许提取更复杂的模式。
