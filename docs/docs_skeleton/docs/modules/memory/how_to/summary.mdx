# 对话摘要记忆

现在让我们来看一下使用稍微复杂一些的记忆类型 - `ConversationSummaryMemory`。这种类型的记忆会随着时间的推移创建对话的摘要。这对于压缩对话中的信息非常有用。
对话摘要内存会实时总结对话并将当前摘要存储在内存中。然后，可以使用该内存将迄今为止的对话摘要注入到提示/链中。对于较长的对话，保留过去的消息历史原样放在提示中会占用太多的标记，因此这种内存非常有用。
